整页详情改造方向

目标已经固定：保留顶部栏和左侧 dock；进入时保留原来的放大动画；流程图只在过渡阶段短暂可见，落定后消失。

A. 共享壳 + 整页详情（推荐）
Topbar
Dock
Map only during transition
True full-page detail content

方案 A

共享一个工作台壳，详情内容改成占满主内容区的真正页面。过渡动画仍由共享壳统一驱动。

优点

  • 最容易保住现有丝滑动画
  • 改动集中，风险最小
  • 视觉上就是“单开页面”

代价

  • 需要把当前方块式容器重构成整页容器
B. 分页彻底拆开
Topbar
Dock
Independent transition layer
Dedicated detail page tree

方案 B

把地图页和详情页彻底拆成两套页面树，再额外做一个跨页面动画层把它们衔接起来。

优点

  • 结构最“正统”
  • 后续扩展整页功能最自由

代价

  • 动画最容易回归
  • 重构量明显更大
C. 原生 View Transition
Topbar
Dock
Browser-driven morph / crossfade
New detail page

方案 C

启用 Next/React 的 View Transition，让浏览器接管跨页面动效，再和现有动画做混合。

优点

  • 理论上最像真正页面切换

代价

  • 实验性更强
  • 和现有 GSAP 过渡容易打架
  • 不适合这次稳改
