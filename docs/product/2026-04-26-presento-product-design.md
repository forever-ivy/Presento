# Presento 产品规划文档

日期：2026-04-26

## 1. 产品定位

Presento 是面向校园演讲、课程答辩、小组展示和比赛路演的 AI 表达训练工具。

它不是通用 AI 聊天工具，不是 AI PPT 生成器，也不是传统 SaaS 后台。Presento 的核心任务是把用户已经拥有的项目资料、PPT、报告、代码、数据和分工信息，转化为一次可以上台表达、可以应对追问、可以复盘改进的训练流程。

产品一句话：

> Presento 帮学生把项目资料练成一次讲得清、答得上、可复盘、可传播的表达。

第一阶段聚焦校园课程项目答辩，因为这个场景高频、真实、痛点明确、Demo 可演示。后续可扩展到：

- 毕业设计答辩
- 小组课堂展示
- 社团宣讲
- 比赛路演
- 实习汇报
- 校园项目短视频介绍

## 2. 核心痛点

目标用户的主要痛点不是“没有内容”，而是“有资料但无法稳定表达”。

具体痛点：

- 资料很多，但讲不清项目目标、技术路线、研究方法或数据来源。
- PPT 已经做完，但没有真正可上台使用的讲稿。
- 项目能跑，但代码、数据库、模型、数据处理等细节答不上。
- 小组分工复杂，用户说不清自己负责范围，也怕被问到队友模块。
- 临近答辩时间紧，缺少高危追问库、兜底回答和训练优先级。
- 练习结束后只知道“答得不好”，不知道薄弱点在哪里、下一轮该练什么。
- 优秀项目表达完后缺少二次传播能力，无法自然沉淀成短视频、图文或项目展示内容。

Presento 要完成的核心转化：

> 从“我有一堆项目资料”到“我能上台讲清楚，并能把表达内容沉淀为可复用、可传播的内容资产”。

## 3. PCG 业务结合

Presento 的 PCG 结合聚焦两条主线：

```text
QQ 校园社交入口
  -> 发起训练、同步分工、提醒进度、分享追问和复盘

微视 / 腾讯视频内容表达出口
  -> 将答辩讲稿和项目介绍转化为短视频口播、分镜和校园 UGC 内容
```

当前 Demo 不需要真实接入 QQ、微视或腾讯视频接口，而是通过模拟入口和产品链路展示业务结合。

### 3.1 QQ：校园协作与训练入口

QQ 是校园学习、小组协作、社团组织和课程沟通的高频场景。课程项目、小组展示、比赛路演通常发生在班级群、课程群、社团群和项目小组群中。

Presento 与 QQ 的结合方式：

- 从 QQ 小组群发起 Presento 训练项目。
- 小组成员上传资料、确认负责范围和答辩顺序。
- 在群内提醒答辩倒计时、今日训练任务和未完成薄弱点。
- 将高危追问、讲稿片段、复盘报告分享到群内，让队友补充问题。
- 将训练成果沉淀为 QQ / QQ 空间可展示的项目摘要或成长记录。

产品中应体现的能力入口：

- 从 QQ 群创建训练
- 同步小组分工
- 分享高危追问到群
- 答辩倒计时提醒
- 分享复盘报告

### 3.2 微视 / 腾讯视频：校园表达内容出口

Presento 不只服务答辩现场，也服务校园表达内容的二次创作。课程项目、社团活动、比赛路演和校园创新项目，都可以从“上台表达”进一步转化为短视频、项目展示片或口播内容。

Presento 与微视 / 腾讯视频的结合方式：

- 将答辩讲稿转成 30 秒 / 1 分钟口播脚本。
- 将项目介绍生成短视频分镜。
- 将模拟讲练中的高质量回答转成视频素材。
- 基于评论区视角生成“观众可能追问”。
- 将答辩复盘转化为内容创作建议。

产品中应体现的能力入口：

- 生成微视口播脚本
- 生成腾讯视频项目展示脚本
- 导出短视频分镜
- 模拟观众追问
- 将复盘转成内容创作建议

### 3.3 非 PCG 产品的处理

ima、腾讯文档、腾讯会议、CodeBuddy 等可作为视觉、交互或技术能力参考，但不作为本作品的 PCG 主业务结合对象。说明文档和 Demo 中如出现这些名称，必须明确其是参考或模拟能力，不应替代 QQ、微视、腾讯视频这两条 PCG 主线。

## 4. 产品核心闭环

Presento 的核心闭环为：

```text
资料导入
  -> 项目知识地图
  -> 逐页讲稿
  -> PPT 同屏语音模拟答辩
  -> 薄弱点钻研
  -> 复盘提升
  -> 内容二次创作
```

这个闭环必须比普通 AI Chat 更明确。用户不是和 AI 闲聊，而是在一个有资料、有节点、有风险、有训练动作的表达工作流中完成准备。

闭环背后由 Presento Agent Skills 驱动。Agent Skills 不是独立卖点，而是可复用、可观察、可测试的 AI 能力单元，用于支撑讲稿生成、代码解释、当前页追问、薄弱点钻研、复盘和内容二次创作。

第一版应同时支持两类 Skills：

- 系统内置 Skills：保证基础训练闭环完整可用。
- 用户自定义 / 导入 Skills：允许用户按自己的专业、课程、老师风格或比赛要求扩展训练能力。

## 5. 核心产品形态

### 5.1 项目知识地图

项目知识地图是 Presento 的核心差异化页面。

它不是文件列表，而是 AI 对项目的理解结果。系统将 PPT、报告、代码、数据库、用户负责范围、高危追问和薄弱点连接为可追踪的知识网络。

新版本的项目知识地图同时承担两个任务：

1. **项目总览**：用 Force-Directed Graph 展示项目结构、资料来源、模块、风险、薄弱点和训练入口。
2. **资料讲解**：当用户点击最末端的资料文件节点时，从图谱总览切换到资料阅读和 AI 讲解状态，帮助用户快速掌握文件内容。

这意味着知识地图不只是“看 AI 理解了什么”，还要成为用户理解资料的入口。它应具备类似 NotebookLM 的文件讲解能力，但讲解目标要始终服务答辩准备，而不是变成通用资料聊天。

资料讲解能力不从零自研交互范式，应优先参考和复用成熟开源 NotebookLM 类项目的实现，例如 Open Notebook、SurfSense 等。可以在许可证允许的范围内直接移植其资料管理、引用回答、来源侧栏、对话会话和内容转换思路，再改造成 Presento 的答辩训练语境。

示例节点：

- 项目中心：智在必得
- 资料节点：PPT、报告、代码、数据库
- 模块节点：AI 模块、智慧书柜、用户端、管理员端
- 风险节点：高危追问、薄弱点、逻辑断层
- 训练节点：查看讲稿、进入讲练、修复薄弱点
- 文件叶子节点：具体 PDF、DOCX、PPT、代码文件、SQL、CSV、XLSX

用户点击任一节点后，应看到：

- 节点说明
- 关联 PPT 页
- 关联代码或资料
- 相关高危追问
- 当前风险等级
- 进入讲练或修复的动作

用户点击文件叶子节点后，应进入资料讲解状态：

- 左侧图谱可收起，保留当前资料在知识图中的位置。
- 中间区域展示资料内容。
- 右侧切换为 AI 讲解卡片，提供摘要、关键概念、答辩风险、证据引用和对话输入。
- 如果资料是 PPT 或被识别为演示稿的 PDF，应优先跳转逐页讲稿页，因为演示材料的核心使用场景是绑定页码生成可上台表达的讲稿。
- 如果资料是普通 PDF 报告、DOCX、代码或 XLSX，应在知识地图页内直接展示内容并支持讲解。

资料讲解支持两种模式：

1. **速通模式**
   面向临近答辩、时间紧的用户。AI 用更短结构帮助用户快速理解文件：一句话总结、必须知道的 3-5 个点、最可能被问的问题、30 秒回答框架和下一步训练入口。

2. **精通模式**
   面向需要真正掌握资料细节的用户。AI 进行分段讲解、概念拆解、来源引用、代码 / 表格 / 数据解释、追问链和自测题，帮助用户把文件内容转化为可讲、可答、可复盘的知识。

不同资料类型的展示方式：

- 普通 PDF：页面预览、页码定位、文本摘录、引用来源。
- DOCX：章节结构、正文阅读、段落引用。
- 代码：类 IDE 阅读体验，支持目录树、文件内容、行号和核心函数解释。
- XLSX / CSV：工作表、表头、字段含义、关键指标和数据口径解释。
- PPT / 演示稿 PDF：跳转逐页讲稿页，按页生成正常版、30 秒版和关键词版讲稿。

### 5.2 逐页讲稿

逐页讲稿将 PPT 页面转化为可上台表达的内容。

每页支持三种版本：

- 正常版：正式展示使用。
- 30 秒版：时间不够时快速讲。
- 关键词版：上台时提词使用。

每页讲稿必须绑定：

- 当前 PPT 页内容
- 项目知识地图节点
- 用户负责范围
- 证据链
- 高危追问

### 5.3 PPT 同屏语音模拟答辩

PPT 同屏语音模拟答辩不是普通聊天框。用户应对着当前 PPT 页面进行口头讲述，AI 老师基于当前 PPT 页、项目知识节点、证据链和用户负责范围，以语音或文字方式连续追问，模拟真实答辩现场。

核心体验是：

```text
用户看着当前 PPT 页面开口讲
  -> AI 老师听取或接收回答
  -> AI 围绕当前页继续追问
  -> 用户请求关键词 / 回答框架 / 参考答案
  -> 卡住的问题进入薄弱点钻研
  -> 训练结束生成复盘
```

训练中应支持：

- PPT 同屏展示。
- 语音回答或文本回答。
- AI 老师语音追问或文字追问。
- 正常追问。
- 高压追问。
- 给关键词。
- 给回答框架。
- 进入薄弱点钻研。
- 生成复盘。

MVP 阶段可以先用文本输入和模拟语音状态表达流程，但页面结构必须为语音答辩预留入口，例如麦克风按钮、听取中状态、AI 老师播报状态和答辩计时。

### 5.4 薄弱点钻研

薄弱点钻研用于修复用户答不上来的问题。

每个薄弱点应包含：

- 来源：模拟讲练、知识地图、代码解释或数据解释。
- 关联节点：PPT 页、代码文件、数据库表、项目模块。
- 风险原因：为什么老师或评委可能追问。
- 证据链：来自哪些资料。
- 修复动作：生成回答框架、加入讲稿、再次讲练。

### 5.5 复盘报告

复盘报告用于训练后的下一步改进。

复盘内容包括：

- 本轮最大风险
- 答不上来的问题
- 回答逻辑问题
- 证据不足的位置
- 新增薄弱点
- 下一轮训练任务
- 可转化为短视频 / 图文的表达内容

### 5.6 内容二次创作

完成答辩训练后，Presento 可将高质量表达内容转化为：

- 30 秒项目介绍
- 1 分钟口播脚本
- 短视频分镜
- 项目展示文案
- 面向观众的 FAQ
- QQ 群 / QQ 空间可分享的项目摘要

这部分是 Presento 与微视 / 腾讯视频内容场景结合的关键。

### 5.7 Agent Skills 系统

Presento 第一版保留 Agent Skills 作为核心 AI 能力层，并支持“系统内置 + 用户自定义 / 导入”的组合模式。

这里的 Skill 不是营销标签，而是一个可复用的表达训练能力单元。每个 Skill 都面向一个明确动作，例如生成逐页讲稿、解释代码、检查数据、模拟老师追问、修复薄弱点或生成复盘。

一个 Skill 必须包含：

- Skill 名称和描述。
- 触发场景：在哪个页面、模式或用户动作下触发。
- 输入信息：项目资料、当前 PPT 页、知识节点、用户回答、项目类型、用户负责范围。
- 能力目标：它要帮助用户完成什么表达训练动作。
- 输出结果：给用户看的结构化内容。
- 来源要求：尽量标明来自哪份资料、哪一页 PPT、哪个代码文件或哪段数据。
- 安全边界：依据不足时说明缺少资料，不编造项目细节。
- 适用范围：项目类型、表达场景或用户角色。

### 5.7.1 系统内置 Skills

MVP 内置 Skills：

1. 项目速记 Skill  
   生成项目一句话介绍、背景、方案、模块、技术路线、数据来源、亮点、分工和最容易被问的问题。

2. 项目知识地图 Skill  
   把 PPT、报告、代码、数据库、分工和风险点整理为项目知识节点和关联边。

3. 逐页讲稿 Skill  
   为每页 PPT 生成正常版、30 秒版和关键词版，并绑定证据链和用户负责范围。

4. 高危追问 Skill  
   生成老师或评委最可能追问的问题，按真实性、技术细节、数据来源、创新点、局限性和分工风险分类。

5. 当前页追问 Skill  
   在 PPT 同屏语音模拟答辩中，基于当前页、用户刚才的回答和知识节点连续追问。

6. 代码 / 数据解释 Skill  
   软件类项目解释目录、核心文件、接口流程、数据库关系和模型流程；调研类项目解释样本、变量、图表、分析方法和局限性。

7. 资料讲解 Skill
   在知识地图中点击文件叶子节点后触发，面向 PDF、DOCX、代码、XLSX / CSV 等资料提供速通和精通两种讲解。输出必须包含文件摘要、关键知识点、证据引用、答辩高危点和可进入讲练或薄弱点钻研的动作。

8. 薄弱点钻研 Skill
   将答不上来的问题拆成知识点，生成证据链、深度追问、学习清单和可加入讲稿的改答版本。

9. 兜底回答 Skill
   在资料不足或问题超出范围时，生成不编造、不慌乱、能承认边界并回到项目依据的回答框架。

10. 复盘报告 Skill
   生成训练总结、风险问题、改答版本、下一轮任务和可回流到知识地图的薄弱点。

11. 内容二次创作 Skill
   将项目讲稿和高质量回答转化为微视 30 秒口播、腾讯视频 1 分钟项目展示脚本、短视频分镜和观众追问。

### 5.7.2 Skill Packs

Skill Packs：

- 软件项目答辩技能包：代码解释、数据库追问、技术选型追问、分工真实性追问。
- AI / 数据项目技能包：数据来源解释、模型效果解释、指标质疑追问、实验复盘。
- 调研报告技能包：问卷设计检查、样本量质疑、图表结论验证、局限性表达。
- 表达冲刺技能包：逐页讲稿、30 秒压缩、关键词提词、兜底回答。
- 内容传播技能包：微视口播、腾讯视频展示脚本、观众追问、标题和简介生成。
- 自定义技能包：用户自己创建或导入的 Skills，可按课程、专业或老师风格组织。

### 5.7.3 用户自定义 / 导入 Skills

用户可以创建或导入自己的 Skills，用于适配更具体的场景。

典型场景：

- 某门课程老师常问的问题风格。
- 某个专业特有的评审标准。
- 某类比赛路演的评分维度。
- 社团宣讲、创业路演、实习汇报等非答辩表达场景。
- 用户自己整理的“答辩追问模板”或“口播脚本模板”。

自定义 Skill 支持两种方式：

1. 表单创建  
   用户填写 Skill 名称、触发场景、输入范围、输出格式、提示词模板和安全边界。

2. 文件导入  
   用户导入 JSON / YAML 格式的 Skill 描述文件。系统校验字段完整性和安全边界后加入个人 Skill 库。

自定义 Skill 的 MVP 边界：

- 允许用户创建、编辑、启用、停用和删除个人 Skills。
- 允许用户导入 JSON / YAML Skill 文件。
- 允许自定义 Skill 使用当前项目资料、PPT 页、知识节点和用户回答作为上下文。
- 不允许自定义 Skill 执行任意代码。
- 不允许自定义 Skill 调用外部网络工具。
- 不开放公开 Skill 市场。
- 不做跨用户共享和付费分发。

这样既保留用户可扩展性，也避免第一版变成复杂插件平台。

## 6. 页面结构

MVP 页面包括：

1. 工作台首页
2. 项目知识地图
3. 资料导入 / 知识源
4. 逐页讲稿
5. 模拟讲练
6. 薄弱点钻研
7. 复盘报告
8. Agent Skills
9. PCG 连接
10. 新建项目

### 6.1 工作台首页

目标：用户进入后知道今天该练什么。

必须展示：

- 当前项目名称
- 准备度
- 高危追问数
- 薄弱点数
- 默认选中的知识节点
- AI 教练建议
- 进入讲练主按钮

首页应采用沉浸式项目星图作为核心视觉，而不是传统卡片 dashboard。

### 6.2 项目知识地图

目标：展示 AI 如何理解项目，并让用户从任一资料节点进入文件讲解和学习。

必须展示：

- 中心项目节点
- 结构节点
- 资料节点
- 文件叶子节点
- AI / 智能模块节点
- 风险节点
- 薄弱点节点
- 训练入口
- 右侧节点详情和证据链

交互要求：

- 左侧或主区域使用专门的 Force-Directed Graph 开源图库表达知识图谱，支持拖拽、缩放、聚焦、节点选中和关联高亮。
- 图谱可以收起或折叠，用户进入资料阅读时仍能看到当前资料所在的知识路径。
- 非文件节点点击后，右侧显示节点说明、关联 PPT、关联代码 / 数据、风险等级、证据链和训练动作。
- 文件叶子节点点击后，图谱态切换为资料阅读态：中间展示资料内容，右侧变成 AI 讲解和对话面板。
- PPT / 演示稿文件点击后进入逐页讲稿页，而不是在知识地图内做普通文件阅读。
- PDF、DOCX、代码、XLSX / CSV 在知识地图页内阅读和讲解。

资料讲解面板：

- 显示当前文件名、资料类型、解析状态和来源位置。
- 支持“速通模式”和“精通模式”切换。
- 速通模式输出：一句话总结、核心要点、答辩必会问题、30 秒回答框架。
- 精通模式输出：章节 / 代码 / 表格拆解、证据引用、深度追问链、自测题和薄弱点候选。
- 支持用户继续提问，AI 回答必须引用当前文件或项目知识库来源。
- 支持将某个问题加入薄弱点钻研、将某段解释加入逐页讲稿或进入模拟讲练。

### 6.3 资料导入 / 知识源

目标：让用户理解资料如何进入知识地图。

必须支持：

- 上传 PPT / PDF
- 上传报告 / 文档
- 上传代码 zip
- 上传 SQL / CSV / Excel
- 录入小组分工
- 模拟从 QQ 群创建训练或导入项目资料

### 6.4 逐页讲稿

目标：让用户把 PPT 变成可讲内容。

必须展示：

- PPT 页缩略图
- 当前页讲稿
- 正常版 / 30 秒版 / 关键词版
- 证据链
- 高危追问
- 进入当前页讲练

### 6.5 模拟讲练

目标：完成对着 PPT 的语音模拟答辩训练。

必须展示：

- 当前 PPT 页面
- AI 老师语音 / 文字追问
- 用户语音回答入口
- 文本回答输入
- 快捷提示
- 当前知识节点
- 倒计时
- 听取中 / 思考中 / 追问中状态
- 讲练结束后的复盘入口

### 6.6 薄弱点钻研

目标：修复训练中暴露的问题。

必须展示：

- 薄弱点队列
- 当前问题拆解
- 证据链
- 推荐回答框架
- 加入讲稿
- 再练一次

### 6.7 复盘报告

目标：给出下一轮训练计划。

必须展示：

- 表达清晰度
- 证据支撑度
- 追问抗压度
- 负责范围准确度
- 新增薄弱点
- 下一轮任务
- 可转成短视频 / 图文的内容建议

### 6.8 Agent Skills

目标：展示 Presento 的 AI 原生能力如何被组织、触发和观察。

必须展示：

- 当前项目启用的 Skill Packs。
- 内置 Skills 列表。
- 用户自定义 Skills 列表。
- 创建 Skill。
- 导入 Skill。
- 每个 Skill 的触发入口。
- 最近 Skill 调用记录。
- 输入来源与输出结果。
- fallback / 依据不足状态。

页面不做公开市场，不展示第三方插件分发。它应表达：

> Presento 通过系统内置 Skills 跑通基础训练闭环，同时允许用户创建和导入自己的 Skills，适配不同课程、专业、老师和比赛场景。

### 6.9 PCG 连接

目标：展示 QQ 与微视 / 腾讯视频业务链路。

必须展示：

- QQ 群训练入口：模拟已创建小组训练。
- QQ 群协作状态：分工、倒计时、训练提醒。
- 微视脚本出口：30 秒口播稿、分镜、观众追问。
- 腾讯视频项目展示出口：1 分钟项目介绍、标题、简介。
- 说明：当前 Demo 为模拟接入。

### 6.10 新建项目

目标：快速创建训练项目。

流程：

```text
选择表达场景
  -> 导入资料
  -> 填写负责范围
  -> 生成项目知识地图
```

表达场景包括：

- 课程项目答辩
- 毕设答辩
- 小组展示
- 比赛路演
- 社团宣讲
- 实习汇报

## 7. AI 原生性

Presento 的 AI 原生性体现在：

- 资料不是简单上传，而是被解析为项目知识地图。
- 讲稿不是单次生成，而是绑定 PPT 页、知识节点和用户负责范围。
- 追问不是普通聊天，而是围绕当前页、证据链和风险连续追问。
- 薄弱点不是人工记录，而是从训练过程自动回流。
- 复盘不是泛泛总结，而是生成下一轮训练任务。
- 内容出口不是简单复制讲稿，而是转化为适合微视 / 腾讯视频的短视频表达内容。
- Agent Skills 不是自由聊天提示词，而是有触发场景、结构化输入输出、来源要求和失败兜底的能力单元。

## 8. MVP 边界

MVP 偏完整产品，而不是只做轻量概念演示。第一版必须跑通完整闭环：

```text
创建项目
  -> 导入资料
  -> 生成知识地图
  -> 生成准备材料
  -> PPT 同屏语音模拟答辩
  -> 薄弱点钻研
  -> 复盘
  -> 内容二次创作
  -> PCG mock 出口
```

MVP 必须做：

- 项目创建：项目名称、表达场景、答辩时间、成员分工、我的负责范围。
- 资料导入：PPT / PDF、报告、代码 zip、SQL / CSV / Excel、小组分工说明。
- 项目知识地图：固定布局或简化星图，支持节点选中、证据链、风险关联和讲练入口。
- 准备材料：项目速记卡、逐页讲稿、高危追问库、代码 / 数据解释。
- PPT 同屏语音模拟答辩：PPT 当前页、麦克风入口、模拟语音状态、AI 老师追问、提示、复盘入口。
- 薄弱点钻研：问题拆解、证据链、改答框架、加入讲稿、再练一次。
- 复盘报告：表达清晰度、证据支撑度、追问抗压度、下一轮任务。
- Agent Skills：内置 Skills、自定义 Skills、Skill Packs、创建 / 导入入口、最近调用记录、fallback 状态。
- 内容二次创作：微视口播脚本、腾讯视频项目展示脚本、短视频分镜、观众追问。
- PCG mock：QQ 群训练入口、分享复盘、微视 / 腾讯视频导出入口。

MVP 不做：

- 真实 QQ / 微视 / 腾讯视频接口接入。
- 多人实时协作。
- 复杂权限系统。
- 真实视频上传发布。
- 完整图数据库。
- 公开第三方 Agent 市场。
- 任意代码执行型 Skill。
- 自定义 Skill 调用外部网络工具。
- 真实 ASR / TTS 可不做，MVP 用模拟语音状态表达流程。

## 9. Demo 演示路径

建议比赛演示路径：

1. 从 QQ 小组群模拟入口创建训练。
2. 上传项目 PPT、代码和报告。
3. 进入项目知识地图，展示 AI 如何理解项目。
4. 点击 AI 模块节点，展示关联 PPT、代码和高危追问。
5. 点击进入 PPT 同屏语音模拟答辩，用户对着当前页开口讲，AI 老师围绕当前页连续追问。
6. 回答卡住后自动生成薄弱点。
7. 进入薄弱点钻研，生成改答框架。
8. 查看复盘报告。
9. 将高质量回答转化为微视 / 腾讯视频口播脚本。

这条路径同时覆盖：

- 赛道适配性
- 作品完整性
- 创新性
- 用户洞察
- AI 原生性
- 落地可行性
- PCG 业务结合
